O Play framework
facilita a criação de
aplicações web com
Java & Scala.

Play é baseado em uma arquitetura leve, stateless e web-friendly com comportamente prevísivel e mínimo consumo de recursos de hardware (CPU, memória, threads) para aplicações escaláveis - graças ao seu modelo reativo, baseado no Iteratee IO.

Poderoso & Rápido

Play aproveita o poder do compilador Scala para fazer o type check de tudo. Afaste-se de erros de execução e usufruia de um desempenho extremo!

Reativo

Play é projetada em torno de um modelo reativo: com base orientada a eventos e sem bloqueio de IO. Prepare-se para o verdadeiro tempo real!

Escalável

Play é baseado em uma arquitetura stateless e web-friendly. É fácil de escalar adicionando mais servidores!


  • Rápido turnaround

    Edite seus arquivos, salve, atualize seu navegador e verá os resultados imediatamente! Não há necessidade de compilar, fazer build, deploy ou reiniciar o servidor.

  • Divertido & Produtivo

    Corte o tempo que você gasta à espera de seu aplicativo para reiniciar, aumente sua produtividade e termine seus projetos mais rápido.

  • Escalabilidade Real

    Escale facilmente executando sua aplicação em vários servidores com Play. Dimensione seu backend em múltiplos núcleos, ou nós múltiplos, usando atores Akka.

  • Java & Scala API

    Use Java, Scala or mix both languages in a single application. Play makes it easier to use the greatest Scala features while keeping 100% compatible with your existing Java code and libraries.

  • Poderoso sistema de build

    Simples, mas extremamente poderoso sistema de construção baseado no SBT. Pronto para uso com tecnologias da Web mais recentes como Less CSS, Google Closure e CoffeeScript.

  • Pronto para usar

    Faça o download do pacote e está pronto. Comece a hackear com o console do Play e o servidor de desenvolvimento. Empurre para a produção com um único comando.

Re-play da história

Desde 2007, temos trabalhado em fazer o desenvolvimento de aplicações web em Java mais fácil. O Play começou como um projeto interno de Zenexity e foi fortemente influenciado pela nossa maneira de fazer projetos web: foco em produtividade do desenvolvedor, respeitando a Arquitetura Web, e desde o início utilizando uma nova abordagem para as convenções de empacotamento.

Desde então, a tecnologia e a Web continuaram a evoluir. A Web tornou-se um ponto central de todas as aplicações. As tecnologias HTML, CSS e JavaScript evoluíram rapidamente - tornando-se quase impossível para um framework server-side acompanhar.

Hoje, toda a arquitetura Web está se movendo rapidamente em direção ao tempo real. Os requisitos do projeto se transformaram também - o que significa conceitos anteriores, como o SQL não são mais a nossa única e exclusivo tecnologia de armazenamento de dados. Temos testemunhado algumas mudanças monumentais em algumas linguagens de programação. A JVM contempla várias linguagens, incluindo Scala, e ganhando popularidade.

Pos isso criamos o Play 2.0, a um novo framework Web para uma nova era.
